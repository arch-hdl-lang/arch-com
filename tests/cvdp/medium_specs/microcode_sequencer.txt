MODULE: microcode_sequencer
ID: cvdp_copilot_microcode_sequencer_0001

=== SPEC ===
Design a SystemVerilog RTL module `microcode_sequencer` that takes a 5-bit instruction as input and generates a 4-bit address for the control store as output. The module decodes the instruction using combinational logic and computes the corresponding control store address based on predefined mappings. This address is an index to access control store entries in an external module, enabling precise microcode execution sequencing. The design ensures low-latency decoding and compatibility with a microcoded architecture.

## Design Specification :
The `microcode_sequencer` RTL module manages the execution of instructions, stack operations, and control signals for a microcoded system. It combines inputs like condition codes, instructions, and data inputs, and enables the generation of outputs for data, stack status, and control signals. The module integrates submodules for stack management, program counter control, instruction decoding, and arithmetic operations.

## Interface:
### Inputs:
- `clk`(1-bit): The positive edge-triggered clock  signal required for the sequential components.
- `c_n_in`(1-bit): Carry-in signal for the ripple carry adder.
- `c_inc_in`(1-bit): Carry-in for the program counter incrementer.
- `r_en`(1-bit): ACTIVE LOW auxiliary register enable signal.
- `cc`(1-bit): ACTIVE LOW condition code input for instruction decoding.
- `ien`(1-bit): ACTIVE LOW instruction enable signal.
- `d_in`(4-bit , [3:0]): A 4-bit data input bus.
- `instr_in`(5-bit, [4:0]): A 5-bit opcode representing the instruction.
- `oen`(1-bit): ACTIVE LOW output enable for the data output path.

### Outputs:
- `d_out`(4-bit,[3:0]): A 4-bit address output bus.
- `c_n_out`(1-bit): Carry-out signal from the ripple carry adder. LOGIC HIGH in this signal indicates the presence of carry.
- `c_inc_out`(1-bit): Carry-out signal from the program counter incremented. LOGIC HIGH in this signal indicates the presence of carry.
- `full`(1-bit): ACTIVE HIGH signal indicates if the Last In First Out (LIFO) stack is full.
- `empty`(1-bit): ACTIVE HIGH signal indicates if the LIFO stack is empty.

## Submodule Overview:
### 1. Stack Management:
#### LIFO Stack: (`lifo_stack`)
- Implements stack operations using submodules for pointer management(`stack_pointer` submodule), memory storage(`stack_ram` submodule), and data selection(`stack_data_mux` submodule).
- Handles push and pop operations based on decoded instructions.
- Provides stack full and empty flags for status monitoring.

  ##### Inputs
  - `clk`(1-bit):  The positive edge-triggered clock signal for controlling reset , incrementing , decrementing of `stack_pointer` and read, write of `stack_ram` locations.
  - `stack_data1_in`(4-bit,[3:0]) : Data coming from the data bus input port `d_in`.
  - `stack_data2_in`(4-bit,[3:0]) : Data coming from the program counter
  - `stack_reset`(1-bit): Control signal coming from the instruction decoder to initialize the stack pointer register to 5'b00000.
  - `stack_push`(1-bit): Used in combination with `full_o` output signal to increment the stack pointer by 1.
  - `stack_pop`(1-bit): Used in conjunction with `empty_o` output signal to decrement the stack pointer by 1.
  - `stack_mux_sel`(1-bit): ACTIVE HIGH control signal coming from the instruction decoder to select one of the two data available at the input of `stack_mux` submodule.
  - `stack_we`(1-bit): Control signal coming from the instruction decoder to write the selected data coming out from the `stack_mux_select` submodule into the `stack_ram` submodule given by the address provided by the `stack_pointer` submodule.
  - `stack_re`(1-bit): Control signal coming from the instruction decoder to read the data from the selected address of the `stack_ram` submodule.
  ##### Outputs :
  - `stack_data_out` (4-bit,[3:0]) - The 4-bit data that is read from a stack location.
  - `full_o` - Indicates whether the `stack_pointer` has reached the highest location of `stack_ram` submodule(i.e. Location number 16).
  - `empty_o` - Indicates whether the `stack_pointer` has reached the lowest location of `stack_ram` submodule(i.e. Location number 0).

   ##### Outputs :

    - `empty`(1-bit): ACTIVE HIGH indicates whether the `stack_pointer` has reached the lowest location of `stack_ram` submodule(i.e. Location number 0). This condition is checked combinationally through continuous assignment statement.

               - `stack_mux_sel` == 1'b0 : `pc_in` is selected.
   ##### Output :
   - `stack_mux_out`(4-bit,[3:0]) : Data selected from one of the two inputs `data_in` or `pc_in`.

##### Interface

### 4. Instruction Decoder: (`instruction_decoder`)
- Decodes the 5-bit instruction opcode and generates control signals for `microcode_arithmetic` , `lifo_stack` `program_counter` and `result_register` modules.
- Supports conditional execution based on condition code (`cc`) input.
##### Interface :
##### Inputs :
- `instr_in`(5-bit,[4:0]) : A 5-bit opcode 
   Supported opcodes :
   
  - `5'b00000 - PRST (Program ReSeT)` - Forces the D output(`d_out`) of the `microcode_sequencer` module to 4'b0000. Loads zero or one into `program_counter` module , depending on the carry input of the `pc_incrementer` submodule of the `program_counter` module and reset  `stack_pointer` submodule of `lifo_stack` module. The contents in locations of `stack_ram` submodule of `lifo_stack` module remains unchanged and the `aux_reg` submodule and `aux_reg_mux` submodules are controlled by `reg_en` input signal of the `microcode_sequencer` module.
   
  - `5'b00001 - Fetch PC` - To provide the content of the `program_counter` module at the D output (`d_out) of the `microcode_sequencer` module.
   
  - `5'b00010 - Fetch R` - To provide the content of the `aux_reg` submodule of the `microcode_arithmetic` module at the D output (`d_out`) of the `microcode_sequencer_module`.
   
  - `5'b00011 - Fetch D` - To provide the content of `d_in` at the D output (`d_out`) of the `microcode_sequencer` module

  - `5'b00100 - Fetch R + D` - To read `d_in` and `aux_reg` contents and provides their sum at the D output (`d_out`) of the `microcode_sequencer` module.
   
  - `5b01011 - Push PC` - To write the contents of `program_counter` module onto the `Top of Stack` given by the content of `stack_pointer` submodule of `lifo_stack` module. This operation also causes increment of the `stack_pointer` submodule
   
  - `5'b01110 - Pop PC` - To read the contents of  `Top of Stack` given by the content of `stack_pointer` submodule of `lifo_stack`  module and provide it at the D output (`d_out`) of `microcode_sequencer` module.

- `cc_in`(1-bit) - Condition code . Generally maintained at LOGIC LOW for the opcodes to be decoded.
- `instr_en`(1-bit) - ACTIVE LOW signal to enable the `instruction_decoder` module to generate control signals.
##### Outputs :
- `cen`(1-bit) : ACTIVE HIGH control signal that allows carry input of `full_adder` submodule of `microcode_arithmetic` module to have effect on addition operation.
- `rst`(1-bit): ACTIVE HIGH control signal that resets the `stack_pointer` register of `stack_pointer` submodule of `lifo_stack` module.
- `oen`(1-bit) : ACTIVE HIGH control signal that enables the output of `full_adder` submodule to be available at D output (`d_out`) of `microcode_sequencer` module.
- `inc`(1-bit) : ACTIVE HIGH control signal that enables the increment operation of `pc_incrementer` submodule of `program_counter` module by 1.
- `rsel`(1-bit) : ACTIVE HIGH control signal LOGICALLY ANDed with ACTIVE LOW `reg_en` signal of `microcode_sequencer` to select a value from `aux_reg_mux` submodule of `microcode_arithmetic` module.
- `rce`(1-bit) : ACTIVE HIGH control signal LOGICALLY ORed with ACTIVE LOW `reg_en` signal of `microcode_sequencer` to transfer the content from input to output of `aux_reg` submodule of `microcode_arithmetic` module.
- `pc_mux_sel`(1-bit) : ACTIVE HIGH control signal to select one of the two data available at the input of `pc_mux`
- `a_mux_sel`(2-bit,[1:0]): Control signal coming from `instruction_decoder` module to select data either from `aux_reg` submodule of `microcode_arithmetic` module , `d_in` coming from the `microcode_sequencer` module or 4-bit constant , 4'b0000.
- `b_mux_sel`(2-bit,[1:0]): Control signal coming from `instruction_decoder` module to select data coming from `aux_reg` submodule or `program_counter` module or `lifo_stack` module or 4-bit constant, 4'b0000.
- `push`(1-bit) : ACTIVE HIGH control signal used along with `full` signal from `stack_pointer` submodule to increment the `stack_pointer` register of `stack_pointer` submodule of `lifo_stack` module by 1.
- `pop`(1-bit):ACTIVE HIGH control signal used along with `empty` signal from `stack_pointer` submodule to decrement the `stack_pointer` register of `stack_pointer` submodule of `lifo_stack` module by 1.
- `src_sel`(1-bit): ACTIVE HIGH control signal  to select one of the two data available at the input of `stack_mux` submodule of `lifo_stack` module.
- `stack_we`(1-bit) : ACTIVE HIGH Write Enable control signal to write the data coming out from`stack_data_mux` submodule into the location generated by `stack_pointer` submodule.
- `stack_re`(1-bit): ACTIVE HIGH Read Enable control signal  to read the data from `stack_ram` location that is provided by `stack_pointer` submodule.
- `out_ce`(1-bit): ACTIVE HIGH control signal to enable the `result_register` module to output the data (`d_out`) from `microcode_arithmetic` module.
   
### 5. Result Register: (`result_register`)
- Captures the output of the arithmetic unit 
- Enables output only when instructed by control signal `out_ce` coming from `instruction_decoder` module.
##### Interface
##### Inputs :
- `clk`(1-bit) : The positive edge-triggered clock signal which controls the data transfer from input to output of `result_register` module
- `data_in`(4-bit,[3:0]) - Data coming from the D output (`d_out`) of `microcode_arithmetic` module.
- `out_ce`(1-bit): ACTIVE HIGH control signal coming from `instruction_decoder` module that is synchronous with `clk` signal to transfer data rom input to output of `result_register` module.
##### Output :
- `data_out(4-bit,[3:0])` : 4-bit data coming out from the `result_register` module.

## Control Flow and Signal Logic:
### Instruction Decoding:
- The instruction decoder interprets the `instr_in` opcode and generates control signals for stack, PC, and arithmetic units.
- Outputs such as `push`, `pop`, `cen`, `rst`, and multiplexer select (`a_mux_sel`, `b_mux_sel`) are driven by the decoded signals.

### Program Counter:
- The PC value is updated based on the instruction, either incrementing or branching to a target address.
- Branching is controlled by the `pc_mux_sel` signal.

### Stack Operations:
- Stack data is pushed or popped based on the `stack_push` and `stack_pop` signals.
- The data source is selected via `stack_mux_sel`, either from the PC or from the output of `microcode_arithmetic` module.

### Data Flow:
- Addition operation is performed using the full adder, with inputs selected by the A(`a_mux` submodule of `microcode_arithmetic` module ) and B multiplexers(`b_mux` submodule of `microcode_arithmetic` module)
- Results are written to the result register or output via `d_out`.

## Key Assumptions and Constraints :
- Instruction and Control Flow:
Based on decoded signals, the module takes a 5-bit instruction opcode (`instr_in`) and determines the appropriate operations for the program counter, stack, and arithmetic unit.
- Pipeline Integration:
This module is designed to function as part of a microcoded pipeline. Based on micro-instructions, it coordinates stack pushes/pops, data flow, and PC updates.
- Behavior:
The module works sequentially, with support of combinational modules such as `full_adder`  , `a_mux` , `b_mux` , `stack_data_mux`  , `pc_mux` submodules. The various submodules of the design are :
  - `lifo_stack` (Submodules present in this are :  `stack_data_mux` , `stack_pointer` , and `stack_ram`)
  - `program_counter` (Submodules present in this are : `pc_mux` , `pc_incrementer` , and `pc_reg`)
  - `microcode_arithmetic` (Submodules present in this are: `aux_reg_mux`, `aux_reg` , `a_mux` , `b_mux` , and `full_adder`) 
  - `instruction_decoder`
  - `result_register` 
- Operating frequency of this module should be 50MHz. The module won't exhibit  correct behavior if there is any deviation in this operating frequency
- Operational Latency :
- The output latency for PUSH and POP instructions will be 2 and 3 clock cycles, respectively. For the remaining instructions, the output will be reflected for the immediate input change without delay.


## Example Scenario:
Instruction: Push PC onto the Stack (`instr_in` = 5'b01011).

The decoder enables `stack_push` and `stack_we`.
The data from the `program_counter` is selected via the multiplexer and written to the stack.

Instruction: Fetch PC (`instr_in` = 5'b00001).

The decoder enables the multiplexers in the `microcode_arithmetic` module, such that the PC value is routed to the Full Adder Output

Instruction: Fetch Auxiliary Register (`instr_in` = 5'b00010).

The decoder configures `a_mux_sel` and `b_mux_sel` for desired inputs.
The value of the auxiliary register is routed to the `full_adder` output
