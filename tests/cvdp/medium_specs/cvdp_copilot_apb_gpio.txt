=== SPEC ===
Design a GPIO module compatible with the Advanced Peripheral Bus (APB) protocol. The module should support configurable GPIO width, bidirectional control, interrupt generation, and robust synchronization. The design should be scalable, well-documented, and free of ambiguities.

---

### **APB Protocol Overview**

The Advanced Peripheral Bus (APB) is a simple, low-latency bus protocol commonly used for connecting peripheral devices in a system. It operates in two phases: 
1. **SETUP Phase**: Signals are configured to initiate a transaction.
2. **ACCESS Phase**: Data transfer occurs when `penable` is asserted.

This design assumes:
- **Always-High Ready**: The `pready` signal remains high, indicating no wait states are introduced.
- **Error-Free Operation**: The `pslverr` signal remains low, indicating no errors occur.

This module supports write and read transactions, aligning with the standard APB protocol behaviors.

---

### **Module Requirements**

#### **1. Module Name and Parameters**
- **Module Name**: `cvdp_copilot_apb_gpio`
- **Parameters**:
  - `GPIO_WIDTH = 8` (default, configurable). Defines the number of GPIO pins.

#### **2. Implementation Language**
The module must be implemented in SystemVerilog, leveraging its advanced features, such as parameterized modules, enumerated types, and enhanced hardware modeling capabilities.

---

#### **3. Input/Output Interfaces**
- **Inputs**:
  - `pclk`: Clock signal (rising-edge triggered).
  - `preset_n`: Active-low asynchronous reset. Initializes all outputs and registers to their reset values.
  - `psel`: APB peripheral select signal.
  - `paddr[7:2]`: APB address bus.
  - `penable`: APB transfer control signal.
  - `pwrite`: APB write control signal.
  - `pwdata[31:0]`: APB write data bus.
  - `gpio_in[GPIO_WIDTH-1:0]`: Input signals from GPIO pins.

- **Outputs**:
  - `prdata[31:0]`: APB read data bus. Reflects the content of the addressed register.
  - `pready`: APB ready signal (always high).
  - `pslverr`: APB error signal (always low).
  - `gpio_out[GPIO_WIDTH-1:0]`: Output signals to GPIO pins, driven by internal registers.
  - `gpio_enable[GPIO_WIDTH-1:0]`: Direction control signals for GPIO pins. **High for output, low for input.**
  - `gpio_int[GPIO_WIDTH-1:0]`: Individual interrupt signals for GPIO pins.
  - `comb_int`: Combined interrupt signal (logical OR of all `gpio_int` signals).

---

#### **4. Register Map**
The module should implement the following registers:
- `0x00`: **GPIO Input Data** (Read-only). Reflects the synchronized state of `gpio_in`.
- `0x04`: **GPIO Output Data**. Controls the `gpio_out` signals.
- `0x08`: **GPIO Output Enable**. Configures the direction via `gpio_enable`.
- `0x0C`: **GPIO Interrupt Enable**. Enables/disables interrupts for individual GPIOs.
- `0x10`: **GPIO Interrupt Type**. Configures edge or level sensitivity for interrupts.
- `0x14`: **GPIO Interrupt Polarity**. Determines **active-high or active-low** interrupt behavior.
- `0x18`: **GPIO Interrupt State** (Read-only). Reflects the current interrupt status.

---

#### **5. Behavioral and Timing Requirements**
- **Clocking and Reset**:
  - The module operates on the **rising edge** of `pclk`.
  - All outputs and internal registers reset to `0` when `preset_n` is asserted (active-low).

- **APB Read/Write Logic**:
  - Decode APB addresses to select the appropriate register.
  - Align `prdata` with the content of the addressed register during read operations.
  - Writes update the corresponding registers on valid address and control signals.

- **GPIO Behavior**:
  - `gpio_out` is driven by the output data register (`reg_out`).
  - `gpio_enable` configures the direction: **High for output, low for input.**
  - Synchronize `gpio_in` with `pclk` using a two-stage flip-flop.

- **Interrupt Logic**:
  - **Edge- or Level-Sensitive Configuration**:
    - The interrupt type is defined in the **GPIO Interrupt Type Register (`0x10`)**.
    - **Edge-Sensitive Mode**: Interrupts are triggered on the rising or falling edge of the input signal, as configured.
    - **Level-Sensitive Mode**: Interrupts are triggered based on the sustained high or low state of the input signal.
  - **Polarity Configuration**:
    - The **GPIO Interrupt Polarity Register (`0x14`)** allows the configuration of **active-high or active-low behavior**, applicable to both edge-sensitive and level-sensitive modes.
  - **Combined Interrupt Signal**:
    - The combined interrupt (`comb_int`) is the logical OR of all individual interrupt signals (`gpio_int`), reflecting the configured edge or level sensitivity.

- **Edge Case Handling**:
  - Undefined APB addresses result in `prdata = 0` and `pslverr = 0`.
  - Handle simultaneous interrupts gracefully by giving priority to lower GPIO numbers.

- **Timing and Latency**:
  - The module introduces no additional latency; all outputs respond within a single clock cycle of the triggering input.

#### **6. Constraints and Edge Cases**
- **Undefined Address Access**:
  - Reads return `0`, writes have no effect.
- **Reset Behavior**:
  - All registers and outputs reset to `0` when `preset_n` is low.
- **Interrupt Generation**:
  - Ensure edge-sensitive interrupts detect both rising and falling edges when configured.
  - Level-sensitive interrupts reflect the sustained state of the input.

import cocotb
from cocotb.triggers import RisingEdge, Timer
from cocotb.clock import Clock
import logging

# Constants
APB_ADDRESSES = {
    'DATA_IN': 0x00,       # Address 0x00 >> 2 = 0x00
    'DATA_OUT': 0x01,      # Address 0x04 >> 2 = 0x01
    'DATA_OUT_EN': 0x02,   # Address 0x08 >> 2 = 0x02
    'INT_ENABLE': 0x03,    # Address 0x0C >> 2 = 0x03
    'INT_TYPE': 0x04,      # Address 0x10 >> 2 = 0x04
    'INT_POLARITY': 0x05,  # Address 0x14 >> 2 = 0x05
    'INT_STATE': 0x06      # Address 0x18 >> 2 = 0x06
}

async def apb_write(dut, address, data):
    """Perform an APB write transaction"""
    # Set APB write signals
    dut.psel.value = 1
    dut.paddr.value = address  # Word address
    dut.pwrite.value = 1
    dut.pwdata.value = data
    dut.penable.value = 0

    # Wait for posedge pclk
    await RisingEdge(dut.pclk)

    # Enable transfer
    dut.penable.value = 1
    await RisingEdge(dut.pclk)

    # De-assert psel and penable
    dut.psel.value = 0
    dut.penable.value = 0

async def apb_read(dut, address):
    """Perform an APB read transaction and return the read data"""
    # Set APB read signals
    dut.psel.value = 1
    dut.paddr.value = address  # Word address
    dut.pwrite.value = 0
    dut.penable.value = 0

    # Wait for posedge pclk
    await RisingEdge(dut.pclk)

    # Enable transfer
    dut.penable.value = 1
    await RisingEdge(dut.pclk)

    # Read prdata
    read_data = int(dut.prdata.value)

    # De-assert psel and penable
    dut.psel.value = 0
    dut.penable.value = 0

    return read_data

def check_condition(condition, fail_msg, pass_msg, test_failures):

    else:

    await RisingEdge(dut.pclk)

    # Retrieve GPIO_WIDTH from DUT parameters
    GPIO_WIDTH = int(dut.GPIO_WIDTH.value)

    cocotb.start_soon(Clock(dut.pclk, 20, units="ns").start())

    dut.preset_n.value = 1

    # Initialize list to collect failures
    test_failures = []

    # Perform APB writes to set up GPIO and Interrupts
    write_data = (0xFFFF >> (32 - GPIO_WIDTH))  # For GPIO_WIDTH=16, 0xFFFF
    write_en = (1 << GPIO_WIDTH) - 1
    await apb_write(dut, APB_ADDRESSES['DATA_OUT'], write_data)
    await apb_write(dut, APB_ADDRESSES['DATA_OUT_EN'], write_en)
    await apb_write(dut, APB_ADDRESSES['INT_ENABLE'], 0x0F)     # Enable lower 4 interrupts
    await apb_write(dut, APB_ADDRESSES['INT_TYPE'], 0x00)       # Level-triggered
    await apb_write(dut, APB_ADDRESSES['INT_POLARITY'], 0x00)   # Active High

    # Clear any pending interrupts
    await apb_write(dut, APB_ADDRESSES['INT_STATE'], (1 << GPIO_WIDTH) - 1)
    await RisingEdge(dut.pclk)

    # Generate Edge on GPIO Inputs
    dut.gpio_in.value = 0x00  # All inputs low
    for _ in range(3):
        await RisingEdge(dut.pclk)

    dut.gpio_in.value = 0xFF  # All inputs high (should generate edges on all bits)
    for _ in range(3):
        await RisingEdge(dut.pclk)

    # Expected interrupts only on lower 4 bits
    expected_int_state = 0x0F & ((1 << GPIO_WIDTH) - 1)
    read_data = await apb_read(dut, APB_ADDRESSES['INT_STATE'])
    actual_int_state = read_data & ((1 << GPIO_WIDTH) - 1)

    check_condition(

        f"FAIL: Interrupts not masked correctly. Expected: 0x{expected_int_state:0{(GPIO_WIDTH + 3) // 4}X}, "

    expected_comb_int = 1 if expected_int_state != 0 else 0

        comb_int == expected_comb_int,

        f"PASS: Combined Interrupt Signal is {'high' if comb_int else 'low'} as expected",
        test_failures
    )

    # Perform APB writes to further test masking
    # Reconfigure Interrupt Polarity to Active High (already active high, but for completeness)

    await RisingEdge(dut.pclk)

    check_condition(

        f"FAIL: Interrupts not masked correctly after re-triggering. Expected: 0x{expected_int_state:0{(GPIO_WIDTH + 3) // 4}X}, "
